# Rename

基于 **Electron + Vue 3 + Vite** 的批量文件重命名桌面应用，在本地选择文件夹后，用可排序的规则组合生成新文件名，并支持预览后再一键重命名。

## 功能概览

- **选择目录**：通过系统对话框选择要处理的文件夹，自动列出其中的文件（不含子目录、不含以 `.` 开头的隐藏文件）。
- **规则积木**：按需添加多种规则块，用 **↑ / ↓** 调整顺序，自由组合命名模板。
- **实时预览**：表格中对比「原文件名」与「新文件名」，确认无误再执行。
- **安全处理**：新文件名中的非法字符（如 `/ \ : * ? " < > |`）会自动替换为 `-`，避免被系统当成路径而报错。

### 支持的规则类型

| 类型 | 说明 |
|------|------|
| **固定文字** | 任意前缀、后缀或分隔符（如 `_`、`-`）。 |
| **时间戳** | 多种格式：如 `YYYYMMDD`、`YYYY-MM-DD`、`YYYYMMDD_HHmmss` 等。 |
| **数字序号** | 可设起始值与补零位数（如 `001`、`002`）。 |
| **原文件名** | 保留不含扩展名的原名。 |
| **图片尺寸** | 读取图片宽高，可自定义宽高之间的连接符（默认 `x`，如 `1920x1080`）。 |
| **图片比例** | 智能匹配常见比例（如 `16:9`、`1:1`），否则用最大公约数化简；比例连接符可自定义（默认 `:`，Windows 下建议改为 `-` 等）。 |
| **文件大小** | 显示为 `KB` / `MB` 等可读字符串。 |

## 技术栈

- [Electron](https://www.electronjs.org/) 29
- [Vue 3](https://vuejs.org/)
- [Vite](https://vitejs.dev/) 5
- [vite-plugin-electron](https://github.com/electron-vite/vite-plugin-electron)

## 环境要求

- **Node.js**：建议 **18.x 或更高**（与当前 Electron / Vite 版本兼容；过旧的 Node 可能无法正常运行 `npm` 或开发命令）。
- 包管理器：可使用 **npm**、**pnpm** 或 **yarn**（仓库中声明了 `pnpm` 作为可选的 `packageManager`）。

## 安装依赖

```bash
cd rename
npm install
# 或
pnpm install
```

## 开发调试

```bash
npm run dev
```

会启动 Vite 开发服务并打开 Electron 窗口，支持热更新。

## 打包发布

```bash
npm run build
```

流程为：先执行 `vite build` 构建渲染进程与主进程产物，再调用 **electron-builder** 打安装包。

- **macOS**：产物在 `release/` 下，常见为 `.dmg`、`.zip`。
- **Windows**：需在 Windows 环境执行同样命令，可得到 `.exe` 安装包与便携版（具体见 `package.json` 中 `build` 配置）。

将对应平台的安装包或压缩包分发给用户即可；用户无需安装 Node。

## 使用步骤（给最终用户）

1. 打开应用，点击 **选择文件夹**。
2. 用 **+** 按钮添加规则，用箭头调整顺序。
3. 在下方表格查看 **新文件名** 预览。
4. 点击 **开始重命名** 批量执行；失败项会在状态中显示。

## 项目结构（简要）

```
rename/

├── index.html

```

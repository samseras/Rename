# Rename

基于 **Tauri + Rust + Vue 3 + Vite** 的批量文件重命名桌面应用，在本地选择文件夹后，用可排序的规则组合生成新文件名，并支持预览后再一键重命名。

由于使用了 Tauri (Rust)，最终打包的安装包体积仅有 **几 MB**，远小于 Electron 的 100MB+，且运行速度极快、内存占用极低。

## 功能概览

- **选择目录**：通过系统对话框选择要处理的文件夹，自动列出其中的文件（不含子目录、不含以 `.` 开头的隐藏文件）。
- **规则积木**：按需添加多种规则块，用 **↑ / ↓** 调整顺序，自由组合命名模板。
- **实时预览**：表格中对比「原文件名」与「新文件名」，确认无误再执行。
- **安全处理**：新文件名中的非法字符（如 `/ \ : * ? " < > |`）会自动替换为 `-`，避免被系统当成路径而报错。

### 支持的规则类型

| 类型 | 说明 |
|------|------|
| **固定文字** | 任意前缀、后缀或分隔符（如 `_`、`-`）。 |
| **时间戳** | 多种格式：如 `YYYYMMDD`、`YYYY-MM-DD`、`YYYYMMDD_HHmmss` 等。 |
| **数字序号** | 可设起始值与补零位数（如 `001`、`002`）。 |
| **原文件名** | 保留不含扩展名的原名。 |
| **图片尺寸** | 读取图片宽高，可自定义宽高之间的连接符（默认 `x`，如 `1920x1080`）。 |
| **图片比例** | 智能匹配常见比例（如 `16:9`、`1:1`），否则用最大公约数化简；比例连接符可自定义（默认 `:`，Windows 下建议改为 `-` 等）。 |
| **文件大小** | 显示为 `KB` / `MB` 等可读字符串。 |

## 技术栈

- [Tauri](https://tauri.app/) (Rust)
- [Vue 3](https://vuejs.org/)
- [Vite](https://vitejs.dev/) 5

## 环境要求

- **Node.js**：建议 18.x 或更高。
- **Rust**：需要安装 Rust 编译环境（`rustc` 和 `cargo`）。
- **C++ 编译工具**：macOS 需要 Xcode Command Line Tools，Windows 需要 Visual Studio C++ Build Tools。

## 安装依赖

```bash
npm install
```

## 开发调试

```bash
npm run tauri dev
```

会启动 Vite 开发服务并编译 Rust 后端，打开 Tauri 窗口，支持热更新。

## 打包发布

```bash
npm run tauri build
```

流程为：先执行 `vite build` 构建渲染进程产物，再调用 `cargo` 编译 Rust 代码并生成安装包。

- **macOS**：产物在 `src-tauri/target/release/bundle/` 下，常见为 `.dmg`、`.app`。
- **Windows**：需在 Windows 环境执行同样命令，可得到 `.msi` 安装包。

将对应平台的安装包或压缩包分发给用户即可；用户无需安装 Node。

## 使用步骤（给最终用户）

1. 打开应用，点击 **选择文件夹**。
2. 用 **+** 按钮添加规则，用箭头调整顺序。
3. 在下方表格查看 **新文件名** 预览。
4. 点击 **开始重命名** 批量执行；失败项会在状态中显示。

## 项目结构（简要）

```
rename/

├── vite.config.js

```
